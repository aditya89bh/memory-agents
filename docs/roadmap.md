# Memory Agents Roadmap

This roadmap defines what is finished, what is planned, and what can be explored later.

---

## Current repo goal

Finish a clear learning path from memory continuity to memory-driven cognition.

This repo is not intended to be a production framework. It is a public research and learning artifact showing how memory systems can be built step by step.

## Project 6 direction: Identity Memory

Identity memory should explore how agents stay consistent across time.

Possible memory types:

- user preferences
- stable agent traits
- long-term goals
- communication style
- boundaries and constraints
- conflict resolution between recent context and stable memory

Minimum proof:

```text
The agent remembers a stable user preference and resolves future responses using that preference.
```

---

## Project 7 direction: Embodied / World Memory

World memory should explore how agents remember environments, states, and action outcomes.

Possible memory types:

- spatial memory
- object memory
- environment memory
- action outcome memory
- failure recovery memory
- robotics task memory

Minimum proof:

```text
The agent remembers that a previous action worked or failed in a specific environment and chooses differently next time.
```

## Future extensions

These should be explored only after Projects 1–7 are clean.

| Area | Possible direction |
|---|---|
| Memory evaluation | Measure retrieval quality, usefulness, and failure modes |
| Multi-agent memory | Shared memory across multiple agents |
| Memory conflict resolution | Handle outdated or contradictory memories |
| Memory decay | Reduce influence of stale memories over time |
| Knowledge graphs | Represent structured relationships between memories |
| Robotics memory | Apply memory systems to physical agents and industrial tasks |
| Memory UX | Design how users inspect, edit, trust, or delete memories |

## Done definition for this repo

The repo is considered finished when:

1. Projects 1–5 are clearly documented.
2. Projects 6–7 are scoped.
3. The root README explains the full progression.
4. The architecture is understandable in under 60 seconds.
5. Each completed project has a clear proof statement.
6. The repo feels like a serious learning path, not a scattered experiment.
